M5 Ink TUI client — chat+dashboard terminal client speaking M4 protocol. Zero imports from domain/, orchestration/, store/, agent-adapter/.

| File | Role | Function |
|---|---|---|
| `index.tsx` | entry | Argv parse → connect → render `<App>`. Enters the alternate-screen buffer (`enterFullscreen`, **writeSync** enter so it lands before Ink's first paint) and enables SGR mouse tracking (`?1000h;?1006h`) for wheel scrolling **and bracketed paste (`?2004h`)** so multi-line pastes arrive wrapped and insert literally. While tracking is on, native text selection / right-click paste need Shift, or Ctrl+T toggles capture off (App writes `?1000l;?1006l`). On EVERY exit (Ctrl+C/onDisconnect, SIGINT/SIGTERM/SIGHUP, fatal, waitUntilExit) it unmounts Ink FIRST, then `leaveFullscreen` (writeSync disables paste `?2004l` + mouse `?1000l;?1006l` + `?1049l`). Handles --resume/--project/--port. Relies on the server handshake's `session.switched`. |
| `App.tsx` | layout | Full-screen layout (root box sized to the terminal via `useStdout`, alt-screen entered in index.tsx) — NO header; Transcript **or** the centered SidePanel (the main row is `justifyContent="center"`; when the dashboard is open the transcript is hidden and the dashboard reads as a horizontally-centered card) + InputBox (which hosts the turn-status line via `statusLine`) + StatusLine + modals. Wires hooks, frame routing. Holds `showShortcuts` (bottom overlay toggled by '?') and `mouseCapture` (default ON for wheel scroll; Ctrl+T / `/mouse` writes the SGR mouse enable/disable sequences to the TTY — OFF frees the mouse for native text selection + right-click paste, stopping the wheel). `handleCommand` maps bespoke slash actions (new/newx/resume/cancel/restart/mouse/help) and **forwards every other registered slash command to the server as its `!<name> <args>` equivalent**. Optimistically echoes genuine user messages (`!`-commands skipped). `/resume <id>` resolves via `matchResumeTarget`. `useMouseScroll` wires wheel → transcript scroll. Status frames (carrying an `actions` rich-block) feed `turnStatus` → InputBox. Computes `focusZone`; `awaitingResponse` blocks send until an agent response. |
| `turn-status.ts` | pure helper | `parseTurnStatus`/`formatTurnStatus` — extract state · time · turns · cost from a status-message text for the one-line status shown tight above the input. |
| `logic.ts` | pure logic | Testable helpers: `computeFocusZone`, `isAgentResponseFrame`, `collectStreamText`, `computeVisibleWindow`, `computeFocusWindow`, `estimateLines`/`computeLineWindow`, `historyPrev`/`historyNext`/`pushHistory` (↑/↓ input-history recall), `matchResumeTarget` (`/resume <id>` resolution), `isMouseSequence`/`parseWheelEvents` (SGR mouse decode + leak guard), `cursorToRowCol`/`rowColToCursor`/`moveCursorVertical` (multi-line input cursor geometry + vertical nav), `stripPasteMarkers`/`normalizeNewlines`/`sanitizePastedText` (bracketed-paste cleanup, incl. bare `[200~`/`[201~`), `classifyDeleteChunk` (Backspace `\x7f` vs forward-Delete `\x1b[3~` from the raw chunk), `detectUserMessage`/`USER_PREFIX` (strip `**You:** `, flag user role), and `wrapToWidth`/`flattenMessageLines`/`flattenTranscript` (line-level flattening; carries a `user` flag per line). |
| `ws-client.ts` | ws class | Typed WS client wrapping M4 protocol. Exponential backoff: 250/500/1k/2k/4k/8k/30s cap. Retry sends resume. Settles to 'disconnected' on backoff-cap so the UI shows the retry hint. |
| `components/Transcript.tsx` | transcript | Bottom-anchored (`justifyContent="flex-end"`, so a short conversation leaves blank at the TOP, never a gap above the input), LINE-level viewport: flattens every message to wrapped display lines (`flattenTranscript`) and slices exactly `lineBudget` lines (`computeVisibleWindow`) so a long message shows in FULL across scroll steps (no truncation) and the slice never overflows. User messages render as a full-width grey-background line (no "You:" prefix; `detectUserMessage` strips it). Scroll = lines (keyboard PgUp/PgDn + mouse wheel); auto-sticks to bottom unless scrolled up. No "N more lines" hints. Empty renders nothing. |
| `components/MessageRow.tsx` | message row (legacy) | Renders text + RichBlock[] + streamed text + queued; `maxChars` truncation. Retained + tested, but Transcript now renders flattened lines directly and no longer uses it. |
| `components/dashboard-constants.ts` | constants | `DASHBOARD_MAX_VISIBLE_ROWS` — per-tab visible-row cap shared by the four list tabs (windowed via `computeFocusWindow`). |
| `components/InputBox.tsx` | input | Custom controlled **multi-line** input. Always typeable; `awaitingResponse` blocks Enter-send while preserving text; `focus` false when dashboard/modal owns the keyboard. Renders the optional `statusLine` (turn status) tight above its border; the old "Press Ctrl+D to return" line was removed (now on StatusLine). **Paste**: bracketed-paste blocks (ESC[200~…) and plain multi-char chunks are sanitized (`sanitizePastedText`) and inserted literally, newlines kept, never submitting. **Multi-line**: trailing `\`+Enter or Alt/Option+Enter insert a newline; render splits on `\n` with the cursor on its (row,col) via `cursorToRowCol`; ↑/↓ move vertically when multi-line else recall input history. **Backspace vs forward-Delete**: Ink reports both as `key.delete`, so they're handled from the raw stdin chunk (Ink's `internal_eventEmitter`) via `classifyDeleteChunk` — `\x7f`→delete-before, `\x1b[3~`→delete-after — not in useInput (which just returns on `key.delete`). '?' on an empty input toggles the shortcuts overlay. Drops mouse residue (`isMouseSequence`). Hosts the `/` slash palette: `SlashMenu` (↑/↓ select, Tab complete, Enter run via `onCommand`, Esc dismiss); **a slash command with args (menu closed) still runs via Enter, forwarding args**; unknown `/word` falls back to `onSubmit`. |
| `components/SlashMenu.tsx` | slash menu | Presentational autocomplete list rendered above the input (filtered `SlashCommand[]` + `selectedIndex`); InputBox owns the keys. |
| `slash-commands.ts` | slash registry | Pure `SLASH_COMMANDS` mirroring the server `!` command set (session: new/newx/resume/cancel/restart; status/orient/projects; cost/budget; tasks/thread/agent/schedule/dispatch; mode/model/backend/profile/skills; devices/nvtop/nvidia-smi/tail; lang/mouse/help) + `parseSlashInput`/`filterSlashCommands`/`findSlashCommand`. App.handleCommand has bespoke actions for new/newx (clear + `!new`/`!newq`), resume (sessions.list → SessionPicker), cancel, restart, mouse (toggle capture), help; **every other command forwards to the server as `!<name> <args>`**. |
| `components/AskUserModal.tsx` | modal | Renders modal.open frames: section/select/multi_select/text_input per M4 spec. ↑/↓ navigate, number keys select, Space toggles multi, Enter confirms. Builds modal.submit values and sends via sendFrame. Displays modal.ack errors inline. |
| `components/PlanFeedbackModal.tsx` | modal | Plan-approval modal variant: plan text (scrollable section) + 3 numbered radio options (Approve/Feedback/Cancel) with hot-keys 1/2/3. Feedback sub-mode for text input. Arrow navigation, Enter/close. App.tsx dispatches when callbackId starts with 'plan'. |
| `components/ConfirmModal.tsx` | confirm modal | Destructive action confirmation — y/Enter confirm, n/Esc cancel. Optional `reasonInput` renders TextInput with `onConfirm(reason)`. |
| `components/StatusLine.tsx` | status bar | Bottom line (the header was removed). Left: `? for shortcuts` hint, OR (when `dashboardActive`) the `Press Ctrl+D to return to the input` hint — moved here off the InputBox. Right: optional `🖱 select` (when `mouseCapture` off — wheel stopped, drag-select on) · project · ⏳ queued · 🔔 notifications. When `showShortcuts` the whole line becomes the full key list (Dashboard/Notifications/Projects/Clear/History/Scroll/**Mouse select**/Cancel/Commands). Connection status is prefixed ONLY when abnormal. |
| `components/SidePanel.tsx` | side panel | Ctrl+D toggle host. Fixed-width (44) box containing Dashboard, **horizontally centered by App's `justifyContent="center"` row** (no marginLeft). Passes `active` (focusZone==='dashboard') so the dashboard owns the keyboard while shown. |
| `components/Dashboard.tsx` | dashboard | Tab-cycled panel: Threads/Tasks/Schedules/Executions/Cost. Tab cycles; Esc closes (`onClose`, mirrors the Ctrl+D toggle) — gated on `active`. Per-tab query/subscribe lifecycle. |
| `components/DashboardThreadsTab.tsx` | threads tab | Thread list with status icon, template name, step progress. ↑/↓ focus, [c] cancel → ConfirmModal → threads.cancel mutate. Inline "(already finished)" feedback on already-terminal error (5s auto-clear). |
| `components/DashboardTasksTab.tsx` | tasks tab | Task list with status/priority/text/claimed. ↑/↓ focus, [c] claim, [u] unclaim, [d] done (ConfirmModal), [b] block (ConfirmModal+reason), [B] unblock → ui.mutate tasks.*. 5s auto-clear error state. |
| `components/DashboardSchedulesTab.tsx` | schedules tab | Schedule list with type/nextRun/paused. ↑/↓ focus, [p] pause, [r] resume, [x] remove (ConfirmModal) → ui.mutate schedules.*. |
| `components/DashboardExecutionsTab.tsx` | executions tab | Execution list with status/type/machine/duration/cost. ↑/↓ focus, [c] cancel (ConfirmModal) → ui.mutate executions.cancel. 5s auto-clear 'not found' feedback. |
| `components/DashboardCostTab.tsx` | cost tab | Cost summary from the `CostSummary` shape: total / this-month / this-week / today USD + per-mode totals (reads real fields `total`/`month`/`week`/`today`/`byMode`, not the old `totalCost`/`models`). |
| `components/Notifications.tsx` | notifications | Corner badge (`🔔 N`) + modal listing active notifications. ↑/↓ navigate, Enter detail, Ctrl+N or Esc close. |
| `components/ProjectSwitcher.tsx` | project switcher | Ctrl+P modal: list projects via ui.query 'projects.list', select to send session.switch. Ctrl+P or Esc closes. |
| `components/SessionPicker.tsx` | session picker | Resumable-session list (--resume startup AND `/resume` mid-session). ↑/↓/Enter select, Esc cancel. Windowed via `computeFocusWindow` (cap 12 + "↑/↓ N more") so a long history can't overflow the terminal; header shows the total count. |
| `hooks/useWsClient.ts` | ws hook | React wrapper around WsClient — connection lifecycle + frame stream. Sends handshake.hello on mount. |
| `hooks/useTranscript.ts` | data hook | Map<messageId, RenderedMessage> for O(1) chat.update. Parallel id[] for insertion order. 30ms batch coalescing for stream.text. `addUserMessage` appends a local `**You:** …` echo flagged `isUser` (server never echoes user input). Orphan stream frames create a synthetic `stream:<id>` message — and never merge into a trailing `isUser` echo, so the reply stays its own row. |
| `hooks/useKeybindings.ts` | key handler | Global key handler via `useInput`. Ctrl+C (dual exits), Ctrl+L clear, PgUp/PgDn scroll, R reconnect. Ctrl+D/N/P toggles, **Ctrl+T toggle mouse capture** (`onToggleMouse`). ↑/↓ are NOT scroll keys — InputBox owns them for input-history recall. Escape is NOT handled here — each active zone owns it. 3rd arg `opts`: `allowScroll`, `allowReconnect`. |
| `hooks/useMouseScroll.ts` | mouse hook | Subscribes to Ink's raw stdin feed (`internal_eventEmitter` 'input'), decodes SGR wheel events (`parseWheelEvents`), and calls onUp/onDown → transcript scroll. Mouse mode is enabled/disabled in index.tsx's fullscreen lifecycle. |
| `hooks/useNotifications.ts` | notif hook | Map<id, NotificationEntry> ring buffer cap 50. Pure helpers `_addNotification`/`_markRead`/`_clearNotifications`. Exposes unreadCount. |
| `hooks/useDashboardData.ts` | dashboard hook | Per-tab state: ui.query on focus, subscribe to events, re-render on ui.event. Pure helpers: `_handleQueryResult`/`_handleEvent`/`_createPendingQuery`/`_clearPendingQuery`. |
| `hooks/useMutate.ts` | mutate hook | Async action hook for ui.mutate request/response — sends mutate frames, matches results by crypto.randomUUID() id, 10s timeout, cleanup on unmount. Returns `{mutate, handleFrame}`. |
| `render/markdown.ts` | md parser | Minimal inline markdown: `**bold**`, `*italic*`, `` `code` ``, `[text](url)`. |
| `render/inline-markdown.tsx` | md renderer | `<InlineMarkdown>` — applies `parseMarkdown` segments as Ink styles (bold/italic/cyan code/underlined link+url), stripping the raw markers. Used by MessageRow + rich-blocks so `**You:**` renders bold instead of literally. |
| `render/rich-blocks.tsx` | block renderer | RichBlock[] discriminated union → `<Box>`/`<Text>` elements; markdown/section/context/default text rendered through `InlineMarkdown`. `actions` renders nothing — the inert `[Resume] [New] [New (quiet)]` buttons were replaced by `/` slash commands. |
